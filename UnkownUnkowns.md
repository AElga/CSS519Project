The following bulletpoints are some unkown unkowns that may apply to my project:

- Normal usage timeslots. What is meant by this is that if some users have emergency appointments outside of the work day time slots as most appointments are (for example, if a patient has an emergency visit at 11pm), then legitimate traffic can happen during hours that may seem not normal. This can broaden what activity times are expected and what times can rase suspicion.
- Changes to API dependencies. APIs that the project relies on can cause the app to break if any updates cause dependency issues. While proper supply chain management could prevent something like this, it is not possible to always know ahead of time what a thrid party developer is implementing/developing with regards to their API
- Changes to patient PII or PHI that is not accounted for. For example, if there are parts of a user's profile that is not easily changeable, problems can occur on PHI if a patient's current information does not match (legal name, marital status, etc.)
- Breaking Updates to HIPAA and other relevant data/privacy laws (unlikely). This unknown unknown is not likey to be impactful as regulations like this don't change too often. However, if data laws suddenly require a new security or tool suite in order to be compliant due to a ground breaking advancment, rapid changes would need to be implemented to ensure compliance.
