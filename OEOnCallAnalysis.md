Given the nature of the Health appointment management application, an "oncall" can use the OE dashboard to narrow down the source of the issue based on metric sources (API, Performance, Database, Security). Once the oncall is able to narrow down the source, they can then watch the logs related to that source to determine a cause. For example, if the oncall determines that there is a Performance Issue based on server health, they can then watch logs/metrics that have to do with server sessions and resources. Then, the oncall can further specify a specific function(s) and/or service(s) that contribute to the issue. For example, the oncall can see that CPU usage is above normal when the appointment booking service is used. From there, any review of how the service is handled can be done to determine what caused the misusage of CPU.
