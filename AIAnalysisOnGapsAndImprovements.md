# Dockerized mock project

Since I have never created my own dockerized project, I had an intergrated in IDE AI help me containerize the mock project step by step. Because of this, I trusted the AI's steps and only confirmed if the docker container commads resulted in the proper running of the mock prototype.

# Threat Model

To create a threat model, I asked ChatGPT to create an outline of a threat model based on the Design Document. From there, I added, removed, and modified the sections to better fit the application I was aiming for.

# Security Test Document

Similar to the threat model, I got an outline from ChatGPT to write the document. However, since I haven't written a Security Test Document, I was more focused on the content rather than the structure that the AI has given me.

# Analysis

While AI was able to get a working version of the dockerized project as well as a structured Threat Model and Security Test Document, I realized that the AI mostly chooses to be vague/abstract with its implementations. For example, the docker.md had simple instructions similar to a templete project. Another example were the placeholder bullet points in the documents (which I modified if I deemed them too vague or not relevent). To improve what the AI has given me, I plan to add a more detailed UI, improve security placeholders, and add some mock data to make the website more functional.
